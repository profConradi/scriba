# Scriba

> **[Versione italiana disponibile qui](README.md)** 🇮🇹

Minimal writing editor with local LLM autocompletion, Markdown support and .docx export.

![License](https://img.shields.io/badge/license-MIT-blue.svg)
![Version](https://img.shields.io/badge/version-1.0.0-green.svg)

## 📖 Overview

Scriba is a **single-file HTML** application (~420 lines) that offers a Markdown text editor with auto-completion suggestions generated by a **local LLM** via **LM Studio**. It requires no server-side: opens directly in the browser and communicates with LM Studio via OpenAI-compatible API on `localhost:1234`.

### Main Features

- ✍️ **Markdown Editor** with auto-resize and monospaced font
- 🤖 **AI Autocompletion** with real-time ghost text
- 📊 **Mathematical Equations** with LaTeX/KaTeX support
- 🔄 **Rephrase and Correct** selected text with AI
- 👁️ **Live Markdown Preview** with professional rendering
- 📄 **Direct .docx Export** (no external conversions)
- 💾 **Load/Save .md** to manage your documents
- ⚙️ **Persistent Configuration** with localStorage
- 🌍 **Bilingual Interface** (Italian/English)

## 🚀 Quick Start

### 1. Install LM Studio

Download and install [LM Studio](https://lmstudio.ai/) for your operating system.

### 2. Load a Model

Open LM Studio and download one of the recommended models (see "Recommended LLM Models" section).

### 3. Configure and Start the Local Server

In LM Studio:
1. Go to **Local Server** (↗️ icon)
2. Load the desired model
3. **IMPORTANT**: Open **Server Settings** (⚙️ top right) and enable **CORS**
4. Click **Start Server**
5. Make sure it's listening on `localhost:1234`

⚠️ **Without CORS enabled, Scriba won't be able to communicate with LM Studio from the browser!**

### 4. Open Scriba

Open `scriba.html` directly in your browser (Chrome, Firefox, Safari, Edge).

The connection indicator in the top right will turn green when Scriba connects to LM Studio.

## 💡 Usage

### AI Autocompletion

Scriba automatically suggests text continuation as you write:

- **Tab** - Accept the suggestion
- **Esc** - Reject the suggestion
- **Ctrl+Space** - Force an immediate suggestion

The ghost text appears in gray italics at the cursor position. The delay is configurable in settings (default: 1.5s).

### Rephrase and Correct

Select text and use the buttons that appear:

- **Rephrase** - Rewrites the text while keeping the meaning (temperature 0.8)
- **Correct** - Fixes grammatical and spelling errors (temperature 0.2)

A diff panel shows the comparison between original and proposal. You can accept or reject.

### Mathematical Equations

Scriba supports LaTeX/KaTeX formulas rendered in preview:

**Inline** (use single dollars):
```markdown
Euler's formula is $e^{i\pi} + 1 = 0$.
```

**Block** (use double dollars):
```markdown
$$
\int_{-\infty}^{\infty} e^{-x^2} dx = \sqrt{\pi}
$$
```

**Systems and Multiline Equations** (use `\\` for line breaks):
```markdown
$$
\begin{cases}
x + y = 5 \\
x - y = 1
\end{cases}
$$
```

### Keyboard Shortcuts

- **Ctrl+P** - Toggle Markdown preview
- **Ctrl+Space** - Force AI suggestion
- **Tab** - Accept ghost text
- **Esc** - Reject ghost text

## 🎯 Recommended LLM Models

For autocompletion you need **low latency** (<2-3s), so **small and fast** models:

### Best for Quality/Speed

| Model | VRAM Required | Notes |
|---------|---------------|------|
| **Qwen2.5-Coder-3B-Instruct** | 4-6 GB | Excellent for code and text, very fast |
| **Qwen2.5-3B-Instruct** | 4-6 GB | Great Italian, good compromise |
| **Phi-3.5-mini-instruct** | 4-6 GB | Microsoft, very fast |

### For More Powerful GPUs

| Model | VRAM Required | Notes |
|---------|---------------|------|
| **Qwen2.5-7B-Instruct** | 8-10 GB | Superior quality, good Italian |
| **Llama-3.2-8B-Instruct** | 10-12 GB | Meta, excellent balance |
| **Gemma-2-9B-Instruct** | 12-16 GB | Google, excellent Italian |

### Reasoning Models (Only for Rephrase/Correct)

⚠️ **Don't use for autocompletion** (too slow):

- **DeepSeek-R1-Distill-Qwen-7B** - Extended reasoning
- **QwQ-32B-Preview** - Advanced thinking mode

Enable **Thinking mode** in settings for these models.

## ⚙️ Configuration

Click **Settings** to customize:

### LLM Parameters

- **Server Address** - LM Studio server URL (default: `http://localhost:1234`)
- **System Prompt** - AI instructions
- **Temperature** - Response creativity (0.0-2.0, default: 0.7)
- **Max Tokens** - Maximum suggestion length (default: 80)
- **Delay (ms)** - Debounce before request (default: 1500ms)
- **Context (chars)** - How many characters to send as context (default: 1500)
- **Language** - Interface language (Italian/English)

### Thinking Mode

For models with extended reasoning (Qwen3, DeepSeek-R1, QwQ):
- Enable **Thinking (extended reasoning)**
- Set **Thinking budget (tokens)** (default: 1024)

⚠️ Increases latency - **not recommended for autocompletion**.

## 📦 Technical Architecture

### Technology Stack

- **Vanilla JavaScript** - No frameworks
- **Marked.js** - Markdown parsing for preview
- **KaTeX** - Mathematical equation rendering
- **JSZip** - .docx file generation
- **FileSaver.js** - File download
- **Google Fonts** - Literata (serif), JetBrains Mono (mono)

### Autocompletion Workflow

1. **Debounce** - After 1.5s pause in typing
2. **Context Extraction** - Last N characters before cursor
3. **LLM Request** - POST to `/v1/chat/completions` (OpenAI-compatible)
4. **Post-processing** - Removal of `<think>` tags and repetitions
5. **Ghost Layer** - Overlay perfectly aligned with textarea

### .docx Export

The `.docx` file is assembled **directly** without external services:

1. **Markdown Parsing** - Conversion to OOXML via `markdownToDocxXml()`
2. **Supported Elements** - Headings, bold, italic, code, lists, quotes
3. **ZIP Assembly** - Complete `.docx` structure with JSZip
4. **Filename** - Extracted from first `# heading` in document

Supported elements:
- Headings `#`-`######` (with decreasing sizes)
- **Bold**, *Italic*, ***Bold+Italic***
- `Inline code` and code blocks
- Bulleted and numbered lists (including nested)
- Quotes `>`
- Horizontal rules `---`

## 🔧 Troubleshooting

### Indicator Remains Red (Disconnected)

- Verify that LM Studio is running
- Check that the local server is active on `localhost:1234`
- **Make sure CORS is enabled in LM Studio** (Server Settings → Enable CORS)
- Go to Settings and verify the server URL
- Check the JavaScript console (F12) for CORS errors

### Suggestions Are Too Slow

- Use a smaller model (e.g. Qwen2.5-3B)
- Disable Thinking mode if active
- Reduce "Context (chars)" in settings
- Verify that GPU is being used in LM Studio

### Suggestions Don't Appear

- Check that the indicator is green (connected)
- Write at least a complete sentence and wait 1.5s
- Press Ctrl+Space to force a suggestion
- Check LM Studio logs for API errors

### Equations Not Rendering

- Equations appear **only in preview** (Ctrl+P)
- Verify LaTeX syntax (use `\\` for line breaks)
- Check console (F12) for KaTeX errors
- In the editor they remain as `$...$` text - this is normal

### HTTP 400 Error from LM Studio

- The model might not support some parameters
- Disable Thinking mode
- Verify that the system prompt doesn't contain special characters
- Check LM Studio logs for details

## 📝 Development Notes

### Code Style

- **Compact** - Short functions on one line where possible
- **camelCase** - For JavaScript variables and functions
- **UI text localized** - Via i18n system
- **No modules** - Everything in global scope of `<script>` tag

### Known Limitations

The .docx export **does not support**:
- Markdown tables
- Images
- Clickable links
- Nested lists beyond 3 levels
- Mathematical equations

To extend, modify `markdownToDocxXml()`.

### External Dependencies (CDN)

All libraries are loaded from CDN for simplicity:
- JSZip 3.10.1
- FileSaver.js 2.0.5
- Marked 15.0.7
- KaTeX 0.16.9
